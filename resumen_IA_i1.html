¿Qué es Machine Learning?

s2_c1 · Denis Parra · Introducción a ML

🧠 Definición Central

💡 Machine Learning es el estudio de técnicas que permiten a los computadores aprender inductivamente, mejorando su rendimiento en una tarea a través de la experiencia (datos), sin ser programados explícitamente para ello.
  • Aprende desde ejemplos particulares → generaliza a nuevos casos
  • Generalmente mejora al exponerse a más datos
  • Busca aprendizaje (representaciones útiles) más que modelación explícita
  • Sólidas bases teóricas, aunque lo práctico suele ir más rápido

🆚 IA Tradicional vs. Machine Learning

IA Tradicional (AGI)

  • Aspiraba a máquinas tan inteligentes como un humano
  • Programación directa de reglas (if-else, lógica)
  • Gran flexibilidad general → muy difícil de construir
  • Ejemplo: ChatGPT parece alinearse con esta visión

Machine Learning

  • Máquinas especializadas en tarea(s) específica(s)
  • Aprenden inductivamente desde datos
  • Pueden superar a humanos en la tarea asignada
  • Depende de la explosión del Big Data (~1990+)
⚠️ Los LLMs (motor de ChatGPT) son construidos con técnicas de ML, pero su comportamiento se acerca a la IA tradicional. Son un punto medio entre ambas concepciones.

El algoritmo de aprendizaje busca en el espacio de hipótesis la hipótesis con menor pérdida en los datos de entrenamiento, apoyándose en el sesgo inductivo del modelo.

La función objetivo general del ML

argmin_W J(X,Y;W) = λ·ℛ(W) + (1/n)·∑ᵢ ℒ(f(xᵢ;W), yᵢ) ───────────────────────────────────────────────────── λ·ℛ(W) → Regularizador (sesgo inductivo, controla complejidad) ∑ℒ(f(xᵢ;W),yᵢ) → Suma de pérdidas en cada ejemplo del training set W → Parámetros del modelo (viven en el espacio de hipótesis)
🔑 Esta fórmula es el hilo conductor de todo el curso. Cada modelo (regresión, SVM, redes neuronales) es una instancia particular de ella con diferente ℛ y diferente ℒ.

Representación de Datos y Generalización

s2_c2 · Denis Parra

📐 Datos como Vectores de Características

  • Cada dato es un vector de características (features / atributos / variables independientes)
  • El conjunto de todos los vectores define el espacio de características (feature space)
  • La dimensionalidad es el número de features → todos los datos deben tener la misma cantidad
  • La etiqueta (label / rótulo) es la variable dependiente: el valor a predecir
  • El identificador del dato nunca es una feature (no aporta información generalizadora)

🗂️ Conjuntos de Datos

ConjuntoUsoNota
Training setEntrenar / ajustar el modeloEl modelo "ve" estos datos
Validation setSeleccionar hiperparámetros y comparar modelosParte del training; no se usa para entrenar inicialmente
Test setEvaluar capacidad de generalización final (holdout)El modelo jamás lo debe ver antes de la evaluación

Validación Cruzada (Cross-Validation)

Usa múltiples y distintos conjuntos de entrenamiento/validación sobre el mismo training set. Caracteriza mejor el rendimiento real, pero es costosa computacionalmente.

⚖️ Bias-Variance Tradeoff

Err(x) = Sesgo² + Varianza + Error irreducible ───────────────────────────────────────────────── Sesgo² = ( E[f̂(x)] − f(x) )² → error sistemático del modelo Varianza = E[ (f̂(x) − E[f̂(x)])² ] → sensibilidad a cambios en los datos Error irreducible = σ² → ruido inherente al problema

Underfitting

  • Modelo demasiado simple
  • Alto sesgo, baja varianza
  • Falla en training Y test
  • Causa: pocas features, modelo poco flexible

Balance óptimo

  • Complejidad adecuada
  • Sesgo y varianza controlados
  • Captura patrones relevantes
  • Buena generalización

Overfitting

  • Modelo demasiado complejo
  • Baja sesgo, alta varianza
  • Bien en training, mal en test
  • Memoriza hasta el ruido
💡 La complejidad del modelo es la palanca que controla el tradeoff. Reducirla → más sesgo; aumentarla → más varianza. El regularizador λ·ℛ(W) es el mecanismo matemático para este control.

Árboles de Decisión

s3_c1 · Denis Parra

🏗️ Estructura y Funcionamiento

  • Técnica de aprendizaje supervisado: sirve para clasificación y regresión
  • Funciona con features binarias, categóricas y numéricas
  • Nodo interno: realiza un test sobre una característica (feature)
  • Arista: resultado del test (valor de la feature)
  • Nodo hoja: clase predicha (clasificación) o valor promedio (regresión)
  • Para clasificar: recorrer desde la raíz hasta una hoja

🔧 Algoritmo de Construcción (Recursivo)

  1. Si todos los registros son de la misma clase → nodo hoja con esa clase
  2. Si todos tienen el mismo valor para todas las features → nodo hoja con la clase más común
  3. De lo contrario:
    1. Seleccionar la mejor feature (mayor Information Gain)
    2. Usarla como nodo raíz del subárbol
    3. Dividir el conjunto y continuar recursivamente por cada rama

Algoritmos clásicos

  • ID3: usa solo entropía, atributos categóricos, divide en 2 ramas
  • C4.5: extiende ID3 para atributos continuos y más de 2 ramas por nodo
  • CART: permite árboles de regresión (minimiza MSE) y clasificación

📐 Criterios de Partición

Entropía

H(D) = −∑_{y∈Y} P(y) · log₂P(y) ───────────────────────────────── • H = 0 → distribución pura (todo en una clase) → ideal • H máx → distribución uniforme → peor caso para clasificar

Ejemplo: 20 datos, mitad clase A y mitad clase B → H = −0.5·log₂(0.5) − 0.5·log₂(0.5) = 1 bit (máxima incertidumbre)

Information Gain (Ganancia de Información)

InfoGain(D, F) = Entropy(D) − ∑_{v ∈ valores(F)} (|Sᵥ|/|D|) · Entropy(Sᵥ) ───────────────────────────────────────────────────────────────────── Elegimos la feature F que maximiza InfoGain → mayor reducción de entropía

Gini Index

Gini(D) = 1 − ∑_{y∈Y} P(y)² Mide la "impureza" o desigualdad entre clases. Alternativa a la entropía.

Para Regresión (CART)

Cada nodo hoja retorna la media de los ejemplos que contiene. El criterio de partición minimiza el error cuadrático (MSE) en cada rama.

Variables numéricas → cómo encontrar el umbral

  • Ordenar por la dimensión, evaluar splits en cada cambio de categoría
  • Elegir el umbral con mayor Information Gain

⚠️ Sobreajuste y Control

🔴 Los árboles tienden a crecer hasta memorizar los datos de entrenamiento → alto riesgo de overfitting. Su sesgo inductivo (Navaja de Occam) favorece árboles poco profundos.

Criterios de parada (Stopping Criteria)

  • Nodos puros (Entropía = 0 o Gini = 0)
  • Se alcanzó la profundidad máxima (max_depth)
  • Mínimo número de muestras por hoja (min_samples_leaf)
  • Sin mejora estadísticamente significativa en la métrica

Técnicas adicionales

  • Detener construcción evaluando en un set de validación
  • Podado (pruning): construir árbol completo y luego eliminar ramas
  • Penalizar complejidad en la métrica de selección del atributo
🪒 Navaja de Occam: "En igualdad de condiciones, la explicación más sencilla suele ser la más probable." Los árboles poco profundos generalizan mejor.

Métodos de Ensamble

s3_c2 · Denis Parra

💡 Idea Fundamental

Combinar múltiples modelos débiles (ligeramente mejor que azar) para producir un modelo fuerte. Condición clave: los modelos deben tener baja correlación entre sí (patrones de error distintos) → los errores se cancelan en el promedio.

🗳️ Analogía: encuesta poblacional. Si preguntamos a suficientes personas independientes, la moda/promedio se acerca a la verdad, incluso si cada individuo se equivoca a veces.

🌲 Bagging → Random Forests

Bagging (Bootstrap Aggregating)

  • Cada árbol se entrena sobre una muestra aleatoria con reemplazo del dataset
  • Reduce la correlación entre árboles → reduce varianza → reduce overfitting
  • Predicción final: voto de mayoría (clasificación) o promedio (regresión)

Random Forests = Bagging + Subconjunto aleatorio de features

  • En cada split, se elige un subconjunto aleatorio de features (típicamente √d features)
  • Evita que una feature muy buena domine todos los árboles → mayor diversidad
  • Limita la profundidad implícitamente → menor complejidad

Evaluación sin set de validación separado

  • Out-of-Bag (OOB) error: cada árbol se evalúa sobre los datos que no vio → estimación gratuita de la generalización
  • Importancia de variables: permutar el valor de una feature entre los ejemplos y medir la caída en performance

📈 Boosting (Reducción Secuencial del Sesgo)

  • Modelos se entrenan secuencialmente, cada uno corrigiendo el error del anterior
  • El modelo n+1 aprende a predecir el residuo (error) del modelo n
  • Reduce el sesgo (bias), a diferencia de bagging que reduce varianza

Variantes modernas (estado del arte en datos tabulares)

  • XGBoost, LightGBM, CatBoost: rendimiento muchas veces superior a redes neuronales en datos tabulares

🏗️ Stacking (Meta-Modelo)

  • Se entrena un meta-modelo que toma como entrada las predicciones de otros modelos previamente entrenados
  • Útil en producción: inferencia liviana con pocas features de entrada
MétodoEstrategiaReduceUso típico
BaggingParalelo, muestras aleatorias de datosVarianzaRandom Forest
BoostingSecuencial, corrige residuosSesgoXGBoost, LightGBM
StackingMeta-modelo sobre prediccionesAmbosCompetencias, producción

📊 Métricas Fundamentales

MétricaFórmulaInterpretaciónCuándo usarla
Accuracy(TP+TN) / Total% de predicciones correctas globalesClases balanceadas
PrecisionTP / (TP+FP)De lo que predigo como positivo, ¿cuánto lo es realmente?Cuando FP es costoso
Recall (Sensibilidad)TP / (TP+FN)De todos los positivos reales, ¿cuántos detecto?Cuando FN es costoso (e.g., diagnóstico médico)

⚠️ Si Recall clase A < Recall clase B, el modelo tiende a predecir más frecuentemente clase B. Puede ocurrir por desbalance de clases en el training set.

📉 Curva ROC y AUC

  • ROC (Receiver Operating Characteristic): grafica TP Rate (eje Y) vs. FP Rate (eje X) al variar el umbral de decisión
  • AUC (Area Under Curve): área bajo la curva ROC. AUC = 1 → clasificador perfecto; AUC = 0.5 → clasificador aleatorio
  • Útil para comparar clasificadores independientemente del umbral elegido
  • Válida principalmente para clasificación binaria

Multiclase → calcular por clase

Para C clases, se calculan C matrices de confusión binarias (una-vs-resto) y se promedian las métricas (macro-average o weighted-average).

Regresión Lineal

s4_c2 · Denis Parra

📐 Formulación del Modelo

f(x; θ) = θ₀ + θ₁x₁ + θ₂x₂ + ... + θₐxₐ = θᵀx (con x₀=1) ───────────────────────────────────────────────────────────── Función de pérdida (MSE — Mínimos Cuadrados): J(θ) = (1/2n) · ∑ᵢ (f(xᵢ;θ) − yᵢ)²
  • Espacio de hipótesis: funciones lineales xᵢ → ℝ
  • Función de pérdida MSE es convexa → existe un mínimo global

🔧 Dos Métodos de Optimización

OLS — Solución Analítica

θ = (XᵀX)⁻¹ Xᵀy
  • Solución exacta, no iterativa
  • Costo: O(n³) por inversión matricial
  • Inviable para datasets muy grandes

Descenso de Gradiente

θ ← θ − α · ∇J(θ)
  • Iterativo; α = learning rate
  • Actualización proporcional al error
  • Escala a datasets grandes
  • En variante SGD: mini-batches

📊 Regresión Polinomial

Se generalizan las features aplicando transformaciones: ϕ(x) = [1, x, x², ..., xᴹ]. Se sigue aplicando regresión lineal sobre las nuevas features transformadas.

⚠️ Grado M muy alto → overfitting (la curva pasa por todos los puntos de entrenamiento pero generaliza mal). Este es el mismo tradeoff de bias-varianza.

🔍 Interpretación de Coeficientes

Ejemplo: Precio = 12M + 45.000·m² + 3.2M·piezas + 2.8M·baños + 4.5M·estac.

  • θ₀ = 12M: precio base (cuando todas las variables = 0, sin interpretación práctica útil aquí)
  • θ₁ = 45.000: cada m² adicional → +$45.000 en precio, manteniendo el resto constante
  • La frase "manteniendo el resto constante" es fundamental: cada coeficiente mide el efecto marginal de esa variable, aislado de las demás

Condiciones para interpretación válida

  • Linealidad: la relación feature→target debe ser aproximadamente lineal
  • No multicolinealidad: features no deben estar altamente correlacionadas entre sí
  • Asociación ≠ Causalidad: θ positivo no implica que la variable cause el aumento del target
  • Si las features están estandarizadas, los coeficientes miden el cambio por desviación estándar

Regresión Logística

s4_c2 · Denis Parra

🔄 De Regresión Lineal a Clasificación

Problema: la regresión lineal puede dar valores fuera de [0,1], no sirve directamente como probabilidad. Solución: aplicar la función logística (sigmoide) a la salida lineal:

σ(z) = 1 / (1 + e⁻ᶻ) donde z = θᵀx ───────────────────────────────────────── P(y=1 | x; θ) = σ(θᵀx) P(y=0 | x; θ) = 1 − σ(θᵀx)
  • Salida en (0,1) → interpretable como probabilidad
  • Decisión: predecir clase 1 si P(y=1|x) ≥ 0.5

📉 Función de Pérdida: Binary Cross-Entropy (BCE)

BCE(θ) = −(1/n) · ∑ᵢ [ yᵢ·log(f(xᵢ;θ)) + (1−yᵢ)·log(1−f(xᵢ;θ)) ] ──────────────────────────────────────────────────────────────────── Regla de actualización (descenso de gradiente): θ ← θ − α · ∇_θ BCE(θ)
💡 La función BCE es la misma entropía cruzada que verás en Deep Learning como función de pérdida para clasificación multiclase. Es uno de los puentes directos entre IA_i1 e IA_i2.

🔍 Interpretación de Coeficientes (Odds Ratio)

Los coeficientes de la regresión logística son lineales en escala de log-odds:

log( P(y=1|x) / P(y=0|x) ) = θᵀx = θ₀ + θ₁x₁ + ... + θₐxₐ ───────────────────────────────────────────────────────────── Odds Ratio (OR) = e^θⱼ → por cada unidad de aumento en xⱼ, los odds se multiplican por e^θⱼ (manteniendo el resto constante)
  • OR > 1 → la variable aumenta las probabilidades del evento
  • OR < 1 → la variable reduce las probabilidades del evento
  • OR = 1 → la variable no tiene efecto

Condiciones para interpretación válida

  • Linealidad en el log-odds para variables continuas
  • No multicolinealidad entre features
  • Independencia de observaciones
  • OR es asociación, no causalidad
  • Muestra suficiente: ~10 eventos por variable incluida

Support Vector Machines (SVM)

s5_c1 · Denis Parra

💡 Idea Central: Máximo Margen

Ante múltiples hiperplanos que separan dos clases con rendimiento perfecto en training, el SVM elige el que tiene el mayor margen (máxima distancia entre el hiperplano y los puntos más cercanos de cada clase).

📐 Intuición geométrica: mayor margen → mayor espacio para datos difíciles o nuevos → mejor generalización.

📐 Álgebra del Hiperplano

Hiperplano: y(x) = β₀ + βᵀx = 0 ───────────────────────────────────────────────────────── • β es ortogonal al hiperplano (demostrado por: βᵀ(xA−xB) = 0 ∀ xA,xB en el hiperplano) • Los coeficientes β indican la orientación del hiperplano • Distancia signada de un punto x al hiperplano: d = (1/‖β‖) · (βᵀx + β₀)

⚙️ Problema de Optimización — Hard-Margin SVM

min_{β,β₀} (1/2) · ‖β‖² sujeto a: yᵢ · (βᵀxᵢ + β₀) ≥ 1 para todo i ───────────────────────────────────────────────────────── Minimizar ‖β‖ es equivalente a maximizar el margen M = 1/‖β‖ Los puntos con yᵢ·(βᵀxᵢ+β₀) = 1 son los VECTORES DE SOPORTE

Resolución mediante el Dual Lagrangiano

  • Condición KKT: αᵢ · [yᵢ·(βᵀxᵢ+β₀) − 1] = 0
  • Si el punto queda sobre el margen: αᵢ > 0 (es vector de soporte)
  • Si el punto queda fuera del margen: αᵢ = 0 (no influye en la solución)
  • La solución solo depende de los vectores de soporte

🔓 Soft-Margin SVM (Datos No Linealmente Separables)

min_{β,β₀,ξ} (1/2)·‖β‖² + C·∑ᵢ ξᵢ sujeto a: yᵢ·(βᵀxᵢ+β₀) ≥ 1 − ξᵢ, ξᵢ ≥ 0 ───────────────────────────────────────────────────────── ξᵢ : variables slack (permiten violar el margen con penalidad) C : hiperparámetro que controla el tradeoff margen/errores
Valor de CMargenErrores permitidosRiesgo
C grande (e.g. 1000)PequeñoPocosOverfitting
C pequeño (e.g. 0.01)GrandeMuchosUnderfitting

Conexión con la fórmula general de ML

J(W) = (1/2)·‖β‖² + C·∑ hinge_loss(yᵢ, f(xᵢ)) ───────────── ───────────────────────────────── Regularizador Función de pérdida (Hinge loss)

El SVM es una instancia perfecta de la forma general argmin λ·ℛ(W) + (1/n)·∑ℒ.

Kernel SVM

s5_c2 · Denis Parra

❓ El Problema de la No-Linealidad

SVM es un clasificador lineal. ¿Qué hacer cuando los datos no son linealmente separables en el espacio original? Solución: transformar el espacio de características a una dimensión mayor donde sí sean linealmente separables.

Problema de la transformación directa: espacios de alta dimensionalidad son costosos en memoria/tiempo, y la distancia euclidiana pierde sentido (maldición de la dimensionalidad).

✨ El Kernel Trick

🪄 Teorema del Kernel: Si la matriz M de productos punto entre vectores transformados h(xᵢ)·h(xⱼ) es positiva definida, entonces existe una función K(xᵢ, xⱼ) que computa ese producto punto directamente en el espacio original, sin necesidad de conocer h(x).
K(xᵢ, xⱼ) = ⟨h(xᵢ), h(xⱼ)⟩ ───────────────────────────────────────────────────────────── Clave: primero definimos K directamente en el espacio de entrada. Luego demostramos que existe algún h() tal que K(xᵢ,xⱼ) = ⟨h(xᵢ),h(xⱼ)⟩. Intuitivamente: un kernel mide la SIMILITUD entre dos vectores en un espacio transformado.

En el problema dual del SVM, los datos solo aparecen como productos punto → basta reemplazar xᵢᵀxⱼ por K(xᵢ,xⱼ).

🔧 Kernels Comunes

KernelFórmulaEspacio implícitoHiperparámetro
LinealK(x,y) = xᵀyOriginal (sin transformación)Ninguno
PolinomialK(x,y) = (xᵀy + c)ᵈPolinomios de grado dd (grado), c
RBF (Gaussiano)K(x,y) = exp(−γ·‖x−y‖²)Dimensión infinitaγ (ancho del kernel)

Efecto de γ en el Kernel RBF

  • γ pequeño → frontera de decisión casi lineal, poco flexible
  • γ grande → frontera muy flexible → riesgo de overfitting
  • Seleccionar γ y C óptimos mediante validación cruzada
✅ SVMs con kernels continúan siendo relevantes y son de los algoritmos off-the-shelf con mejor rendimiento. Han perdido popularidad frente a Deep Learning y ensambles en la última década.

📋 Resumen Ultra-Compacto por Tema

🤖 ML: aprendizaje inductivo desde datos. Marco general: argmin J = λℛ + (1/n)∑ℒ. Tipos: supervisado, no supervisado, reforzado.
📦 Datos: vectores de features + etiqueta. Train/Val/Test. Bias-Variance: underfitting (alto sesgo) vs. overfitting (alta varianza). Regularizador controla el tradeoff.
🌳 Árboles: construcción recursiva maximizando Information Gain (entropía). Sesgo inductivo: árboles simples (Occam). Gran riesgo de overfitting → poda, max_depth.
🌲 Ensambles: Bagging (RF, reduce varianza, paralelo), Boosting (XGBoost, reduce sesgo, secuencial), Stacking (meta-modelo). Modelos con baja correlación entre sí = clave.
📏 Métricas: Accuracy (clases balanceadas), Precision/Recall (costos asimétricos), F1 (compromiso P-R), ROC-AUC (evaluación global del clasificador binario).
📈 Reg. Lineal: espacio lineal, pérdida MSE (convexa). OLS: θ=(XᵀX)⁻¹Xᵀy. Grad. Desc.: θ←θ−α∇J. Coeficientes = efecto marginal manteniendo el resto constante.
🎲 Reg. Logística: sigmoide sobre salida lineal → probabilidad. Pérdida BCE. Coeficientes en log-odds → exponenciar para OR. Clasificación binaria.
⚔️ SVM: hiperplano de máximo margen. Hard-margin (lineal sep.), Soft-margin (C controla margen/errores). Solo los vectores de soporte definen la solución.
🔮 Kernel SVM: K(x,y) = ⟨h(x),h(y)⟩ sin calcular h(x). RBF → espacio infinito. γ controla flexibilidad. Kernel trick = operar en espacio de alta dimensión barato.
