🔵 Active Learning

Resumen conceptual

Active Learning surge cuando etiquetar datos es costoso. El modelo elige cuáles ejemplos vale la pena etiquetar (consulta al oráculo) para maximizar el aprendizaje con el mínimo de etiquetas.

ConceptoDescripción
Pool-based ALHay un pool U de datos sin etiqueta. Se elige el más informativo para consultar.
OráculoEntidad que responde la etiqueta correcta (humano experto).
L / UConjunto etiquetado (labeled) / sin etiquetar (unlabeled)
Cold startProblema inicial: sin datos etiquetados no se puede entrenar un modelo inicial.
Distributional shiftSi U no es representativo de la distribución real, el modelo puede sesgarse.

Estrategias de consulta

EstrategiaIdeaCuándo usar
Uncertainty / Least ConfidentConsultar el ejemplo con menor confianza máxima: 1 - P(ŷ|x)Clasificación binaria o multiclase
Margin SamplingMenor diferencia entre la 1ª y 2ª clase más probableCuando hay clases "competidoras"
Entropy SamplingMáxima entropía: -Σ P(y|x)·log P(y|x)Multiclase, captura toda la incertidumbre
Query by Committee (QbC)Comité de modelos; consultar donde más desacuerdo hayCuando se pueden entrenar múltiples modelos
Core-SetSeleccionar puntos que maximizan cobertura del espacio (min-max-distance)DL, cuando la función de pérdida no da buena incertidumbre
Batch ModeSeleccionar B ejemplos a la vez (no uno a uno)Cuando etiquetar en batch es más eficiente

Ejercicios Active Learning

AL-1Básico

¿Qué es Active Learning y por qué es útil? Describe el ciclo básico de pool-based active learning y los 3 componentes principales.

Active Learning es un paradigma de aprendizaje supervisado donde el modelo puede consultar activamente las etiquetas de ejemplos seleccionados del pool sin etiquetar. Es útil cuando etiquetar datos es caro (médicos, expertos, etc.).

Ciclo pool-based:

  1. Entrenar modelo con conjunto etiquetado L (inicialmente pequeño).
  2. Aplicar estrategia de consulta para seleccionar el ejemplo más informativo de U.
  3. Consultar al oráculo por su etiqueta.
  4. Agregar el ejemplo etiquetado a L y repetir.

3 componentes: (1) Modelo de aprendizaje, (2) Estrategia de consulta, (3) Oráculo.

AL-2Básico

Least Confident: Un clasificador da las siguientes probabilidades para 4 ejemplos sin etiquetar (clasificación binaria, clases {0,1}):

EjemploP(1|x)P(0|x)
x10.950.05
x20.510.49
x30.700.30
x40.600.40

¿Cuál elegiría Least Confident? ¿Y Margin Sampling? ¿Coinciden?

Least Confident = elige el de menor P(ŷ|x) = 1 - max(P(y|x)):

  • x1: 1 - 0.95 = 0.05
  • x2: 1 - 0.51 = 0.49 ← máximo
  • x3: 1 - 0.70 = 0.30
  • x4: 1 - 0.60 = 0.40

Margin Sampling = elige el de menor diferencia entre 1ª y 2ª clase:

  • x1: 0.95 - 0.05 = 0.90
  • x2: 0.51 - 0.49 = 0.02 ← mínimo margen
  • x3: 0.70 - 0.30 = 0.40
  • x4: 0.60 - 0.40 = 0.20

Ambos eligen x2. Esto siempre ocurre en clasificación binaria porque Margin = |P(1) - P(0)| = |2·P(ŷ) - 1|, que es mínimo cuando P(ŷ) es mínimo.

AL-3Intermedio

Entropy Sampling: Un clasificador de 3 clases da las siguientes probabilidades:

EjemploP(c1|x)P(c2|x)P(c3|x)
x10.80.10.1
x20.340.330.33
x30.50.40.1

Calcula la entropía de cada ejemplo y elige cuál consultar. Recuerda: H = -Σ p·log₂(p).

H(x1) = -(0.8·log₂(0.8) + 0.1·log₂(0.1) + 0.1·log₂(0.1))
= -(0.8·(-0.322) + 0.1·(-3.322) + 0.1·(-3.322))
= -(−0.257 − 0.332 − 0.332) = 0.922 bits

H(x2) = -(0.34·log₂(0.34) + 0.33·log₂(0.33) + 0.33·log₂(0.33))
≈ -(0.34·(-1.556) + 0.33·(-1.599) + 0.33·(-1.599))
≈ -(−0.529 − 0.528 − 0.528) = ≈1.585 bits (≈ máximo para 3 clases)

H(x3) = -(0.5·log₂(0.5) + 0.4·log₂(0.4) + 0.1·log₂(0.1))
= -(0.5·(-1) + 0.4·(-1.322) + 0.1·(-3.322))
= -(−0.5 − 0.529 − 0.332) = 1.361 bits

Se elige x2 (mayor entropía = mayor incertidumbre). Tiene distribución casi uniforme entre las 3 clases.

AL-4IntermedioAyudantía 5

Query by Committee: Tienes un comité de 4 modelos que predicen las clases de 3 ejemplos:

M1M2M3M4
x1AAAB
x2ABAB
x3BBBB

Usando vote entropy como medida de desacuerdo, ¿cuál ejemplo se consulta?

Vote entropy: H = -Σ (V(y)/|C|) · log(V(y)/|C|), donde V(y) = votos para clase y, |C| = total modelos.

  • x1: A=3/4, B=1/4 → H = -(0.75·log(0.75) + 0.25·log(0.25)) = 0.811 bits
  • x2: A=2/4, B=2/4 → H = -(0.5·log(0.5) + 0.5·log(0.5)) = 1.0 bit ← máximo
  • x3: B=4/4 → H = 0 (unanimidad)

Se elige x2 — máximo desacuerdo entre modelos (50-50).

AL-5Difícil

Limitaciones de Active Learning: Explica los problemas de (a) cold start, (b) distributional shift, y (c) por qué Uncertainty Sampling puede fallar en redes neuronales profundas.

(a) Cold start: Sin ejemplos etiquetados iniciales (L = ∅), no se puede entrenar ningún modelo para aplicar la estrategia de consulta. Solución: comenzar con algunas etiquetas aleatorias o usar clustering para seleccionar ejemplos representativos.

(b) Distributional shift: Si el pool U no representa la distribución real de test, el modelo aprende bien sobre U pero generaliza mal. La selección activa puede exacerbar este bias: si el modelo siempre consulta en una región del espacio, ignora otras regiones importantes.

(c) Uncertainty Sampling en DL: Las redes neuronales pueden dar probabilidades muy confiadas (alta P(ŷ)) incluso en ejemplos fuera de la distribución de entrenamiento, gracias al softmax. Un punto lejano a todos los datos de entrenamiento puede recibir P(ŷ) = 0.99. Por eso no es un buen indicador de incertidumbre real. Alternativas: Monte Carlo Dropout (estimar incertidumbre epistémica), Core-Set (geométrico, no depende de probabilidades).

AL-6Nivel ExamenAyudantía 8

Comparación de estrategias: Tenemos un clasificador binario sobre un espacio 2D. Los puntos en U son:

  • Región A (frontera de decisión): P(1|x) ≈ 0.52, puntos densamente agrupados
  • Región B (lejos de datos): P(1|x) ≈ 0.95, punto aislado fuera del soporte de los datos

Argumenta qué estrategia elegiría cada punto y cuál es más útil para el aprendizaje.

Uncertainty Sampling: Elegiría Región A (P(1|x)=0.52, incertidumbre ~0.48). La Región B tiene alta confianza (score bajo = 0.05). ✓ Correcto en este caso.

Core-Set: Elegiría Región B si está lejos del centroide de los datos ya etiquetados (maximiza cobertura del espacio de features). Puede ser útil para representar mejor el espacio, pero si ese punto está realmente fuera de la distribución de interés, es una pérdida de presupuesto.

¿Cuál es más útil? Depende del objetivo: si queremos refinar la frontera de decisión, Uncertainty Sampling (Región A) es mejor. Si queremos cubrir mejor el espacio (evitar distributional shift), Core-Set puede ser mejor. En la práctica, Región A es la respuesta correcta porque impacta directamente la frontera de decisión, que es lo que determina la accuracy del clasificador.

🟠 Reinforcement Learning

Resumen conceptual: MDP y Q-learning

Un MDP (Markov Decision Process) se define como (S, A, r, p, γ):

SímboloSignificado
SConjunto de estados
AConjunto de acciones
r(s,a,s')Recompensa al pasar de s a s' con acción a
p(s'|s,a)Probabilidad de transición
γ ∈ [0,1]Factor de descuento (qué tanto importa el futuro)
π(s)Política: qué acción tomar en cada estado
V(s)Valor esperado del retorno al estar en s siguiendo π
Q(s,a)Valor esperado del retorno al tomar a en s
Ecuación de Bellman: Q(s,a) = E[r + γ · max_a' Q(s',a')]

Q-learning update: Q(s,a) ← Q(s,a) + α · [r + γ · max_a' Q(s',a') − Q(s,a)]

Donde: α = learning rate, γ = descuento, r = recompensa inmediata

ε-greedy exploration

Con prob. ε: acción aleatoria (exploración). Con prob. 1-ε: mejor acción según Q (explotación).

Ejercicios Reinforcement Learning

RL-1Básico

Define los componentes de un MDP. ¿Qué distingue a RL de aprendizaje supervisado?

MDP: (S, A, r, p, γ). El agente observa un estado s, toma una acción a, recibe una recompensa r y transiciona a s'.

Diferencia clave: En aprendizaje supervisado, hay un "teacher" que da la respuesta correcta para cada ejemplo. En RL, solo hay una señal de recompensa (puede ser rara, tardía y ruidosa). El agente debe descubrir por sí mismo qué acciones son buenas mediante exploración. No hay un "correcto" instantáneo — la acción buena puede llevar a una recompensa recién muchos pasos después (crédito temporal).

RL-2Básico

¿Qué significa γ (factor de descuento)? ¿Qué sucede con γ=0 y γ=1?

γ pondera la importancia de las recompensas futuras relativo a las inmediatas.

γ = 0: El agente es completamente miope — solo le importa la recompensa inmediata. Política greedy de corto plazo. Problema: puede tomar acciones que arruinan el futuro.

γ = 1: El agente valora igual recompensas presentes y futuras (suma no descontada). Problema: puede no converger si los episodios son infinitos y hay ciclos.

Valor típico: γ = 0.9 ó 0.99 — equilibrio entre horizonte temporal y estabilidad.

RL-3Intermedio

Cálculo Q-learning: Considera un MDP con 2 estados (s1, s2) y 2 acciones (a1, a2). La Q-table actual es:

a1a2
s10.50.3
s21.00.8

El agente está en s1, toma a2, recibe r=1.0, y llega a s2. Actualiza Q(s1,a2) con α=0.1, γ=0.9.

Fórmula: Q(s,a) ← Q(s,a) + α·[r + γ·max_a' Q(s',a') − Q(s,a)]

max_a' Q(s2, a') = max(1.0, 0.8) = 1.0

Target: r + γ·max Q(s2) = 1.0 + 0.9·1.0 = 1.9

Error (TD): 1.9 − Q(s1,a2) = 1.9 − 0.3 = 1.6

Nueva Q(s1,a2) = 0.3 + 0.1·1.6 = 0.3 + 0.16 = 0.46

El Q sube porque la recompensa (1.9) fue mayor de lo esperado (0.3). El error TD positivo indica que la acción fue mejor de lo que el modelo pensaba.
RL-4IntermedioAyudantía 6

ε-greedy: ¿Por qué es necesario explorar? ¿Qué pasa si ε=0 (solo explotación) desde el inicio?

Por qué explorar: La Q-table se inicializa con valores arbitrarios (típicamente 0). Si solo explotamos desde el principio, el agente siempre tomará la primera acción que parezca buena según esos valores iniciales, sin nunca descubrir que otras acciones pueden ser mejores. Se queda atrapado en un óptimo local.

ε=0 desde el inicio: Si Q(s,a)=0 para todo (s,a) y el agente usa ε=0, elegirá la primera acción en el orden de desempate y nunca la cambiará. La Q-table se actualizará solo para esa trayectoria, dejando todo el resto inexplorado.

Solución: Empezar con ε alto (mucha exploración) y decaerlo gradualmente (ε-decay). Así el agente explora al inicio y explota cuando ya tiene buenas estimaciones de Q.

RL-5Difícil

Convergencia y Deep Q-Networks: ¿Por qué Q-learning con Q-table no escala a espacios de estado grandes? ¿Qué introduce DQN y por qué mejora la estabilidad?

Problema de escala: Con una Q-table, necesitamos guardar Q(s,a) para cada par (s,a). Si el espacio de estados es continuo o muy grande (ej. el píxel-space de un videojuego: 84×84×4 = 28,224 dimensiones), la tabla es imposible de mantener o llenar.

DQN: Usa una red neuronal Q(s,a;θ) que aproxima los Q-values. Dado el estado s como entrada, produce Q(s,a) para todas las acciones a simultáneamente.

Dos trucos de estabilidad:

  • Experience Replay: Guarda transiciones (s,a,r,s') en un buffer. En cada paso, entrena con un mini-batch aleatorio del buffer. Rompe correlaciones temporales y mejora la eficiencia de datos.
  • Target Network: Usa una red "congelada" θ⁻ para calcular el target (r + γ·max Q(s',a';θ⁻)). Actualiza θ⁻ cada C pasos. Sin esto, el target cambia con cada update, haciendo el entrenamiento inestable.
RL-6Nivel ExamenAyudantía 8

MDP completo: Un robot puede estar en 3 estados: A, B, C (meta). En cada estado puede ir Izquierda (I) o Derecha (D). Transiciones determinísticas: A→I=A, A→D=B, B→I=A, B→D=C, C es terminal. Recompensas: llegar a C = +10, cualquier otra transición = -1. γ=0.9.

Calcula Q(A,D) y Q(A,I) en estado estacionario (política óptima).

El camino óptimo desde A es: A→D→B→D→C, con recompensas -1, -1, +10.

Q(B,D): Tomar D desde B lleva a C con r=+10. Estado terminal → V(C)=0.
Q(B,D) = 10 + γ·V(C) = 10 + 0.9·0 = 10

Q(B,I): Tomar I desde B lleva a A con r=-1.
V(A) = max(Q(A,I), Q(A,D)) — lo calcularemos.
Q(B,I) = -1 + 0.9·V(A)

Q(A,D): Lleva a B con r=-1.
V(B) = max(Q(B,I), Q(B,D)) = max(Q(B,I), 10) = 10
Q(A,D) = -1 + 0.9·10 = -1 + 9 = 8

Q(A,I): Lleva a A con r=-1 (ciclo).
V(A) = max(Q(A,I), Q(A,D)) = max(Q(A,I), 8)
Q(A,I) = -1 + 0.9·V(A) = -1 + 0.9·8 = -1 + 7.2 = 6.2

Verificación: V(A) = max(8, 6.2) = 8. La política óptima: en A → D, en B → D.

Capa 1 (ReLU): W₁=[[1,-1],[2,0]], b₁=[0,1]

DL-5DifícilAyudantía 7

Backpropagation: Para la red del ejercicio DL-3, calcula ∂L/∂W₂ si L = CE = -log(ŷ) con ŷ=0.993 y la clase correcta es 1.

L = -log(ŷ) → ∂L/∂ŷ = -1/ŷ ≈ -1/0.993 ≈ -1.007

σ'(z) = σ(z)·(1-σ(z)) = 0.993·0.007 ≈ 0.00695

∂L/∂z₂ = ∂L/∂ŷ · ∂ŷ/∂z₂ = -1.007 · 0.00695 ≈ -0.007 (esto es δ₂)

∂L/∂W₂ = δ₂ · h₁ᵀ = -0.007 · [1, 3] = [-0.007, -0.021]

Interpretación: Los gradientes son pequeños porque la red ya predice bien (0.993) — sigmoid satura, gradiente casi nulo.

Este es el problema de vanishing gradient con sigmoid en redes profundas. ReLU no satura para valores positivos, por eso se prefiere en capas ocultas.
DL-6Difícil

Regularización: Explica dropout y weight decay. ¿Cómo reducen el sobreajuste? ¿Se usan en test?

Dropout: Durante entrenamiento, cada neurona se "apaga" con probabilidad p (típico p=0.5). Efecto: la red no puede depender de ninguna neurona individual → aprende representaciones más robustas y distribuidas. En test: se desactiva el dropout y se escalan los pesos por (1-p) para compensar. No se usa en inferencia.

Weight decay (L2 regularización): Agrega λ||W||² a la función de pérdida. En el gradiente: ∂L/∂W incluye un término +λW que "empuja" los pesos hacia cero. Penaliza pesos grandes, prefiriendo soluciones más simples. Equivale a prior Gaussiano sobre los pesos (Bayesiano). Sí afecta el entrenamiento, pero en test no se necesita ajuste.

En test: Dropout OFF, weight decay no activo (solo afecta la función de pérdida durante el entrenamiento).

🟢 Redes Neuronales Convolucionales (CNNs)

Fórmulas clave

Dimensión de salida de una convolución: H_out = (H_in - F + 2P) / S + 1 W_out = (W_in - F + 2P) / S + 1 Parámetros de una capa conv: Params = F × F × C_in × K + K (con bias) Donde: F=tamaño filtro, C_in=canales entrada, K=cantidad de filtros Max Pooling (sin parámetros): H_out = (H_in - P_size) / stride + 1

CNN-5DifícilAyudantía 7

¿Por qué CNNs en vez de MLPs para imágenes? Explica los 3 principios clave de diseño de una CNN y cómo cada uno ayuda.

1. Localidad (filtros locales): Los patrones visuales (bordes, texturas) son locales — un píxel en la esquina superior no depende de uno en la esquina inferior. Filtros de 3×3 ó 5×5 capturan patrones locales, reduciendo drásticamente parámetros vs. fully connected.

2. Weight sharing (compartición de pesos): El mismo filtro se aplica a toda la imagen. Un detector de bordes horizontales funciona en cualquier parte de la imagen. Esto implementa invarianza traslacional y reduce parámetros en un factor del tamaño de la imagen.

3. Jerarquía de features: Capas iniciales detectan patrones simples (bordes, colores). Capas profundas combinan estos en patrones más complejos (ojos, ruedas, caras). Similar a la corteza visual (Hubel y Wiesel, 1960s).

Resultado: Una CNN para imágenes 256×256 tiene órdenes de magnitud menos parámetros que un MLP equivalente, generaliza mejor, y respeta la estructura espacial de los datos.

CNN-6Nivel Examen

AlexNet y transfer learning: Explica por qué AlexNet (2012) fue un hito. ¿Qué técnicas introdujo? ¿En qué consiste transfer learning y cuándo es útil?

AlexNet fue un hito porque:

  • Usó GPUs para entrenamiento (primer trabajo a gran escala)
  • Introdujo ReLU en vez de sigmoid/tanh → entrenamiento 6× más rápido
  • Usó Dropout para regularización
  • Data augmentation (5 crops × 2 flips = 10× más datos)
  • Entrenó sobre ImageNet (1.2M imágenes) — escala sin precedentes
  • Error top-5 de 15.3% vs 26% del segundo lugar → brecha enorme

Transfer learning: Tomar una red preentrenada en un dataset grande (ImageNet) y re-usarla para una tarea nueva con pocos datos. Las capas iniciales aprenden features genéricas (bordes, texturas) transferibles a cualquier dominio visual.

Estrategias: (a) Feature extractor: congelar capas conv, entrenar solo el clasificador final. (b) Fine-tuning: descongelar capas más profundas y re-entrenar con learning rate bajo.

Cuándo usar: Cuando tienes pocos datos de la tarea objetivo pero hay un dataset grande relacionado. Evita sobreajuste y acelera convergencia.

🔴 Razonamiento Deductivo

RD-1Básico

¿Qué es razonamiento deductivo? Distingue Sistema 1 vs. Sistema 2 (Kahneman).

Razonamiento deductivo (Russell & Norvig): Proceso por el cual se derivan afirmaciones que se concluyen lógicamente a partir de una base de conocimiento (premisas).

Sistema 1: Rápido, automático, sin esfuerzo consciente. Ej: reconocer una cara, leer texto familiar.

Sistema 2: Lento, deliberado, con esfuerzo consciente. Ej: resolver 17×24, demostrar un teorema, jugar ajedrez.

El razonamiento deductivo es Sistema 2 — requiere seguir reglas lógicas paso a paso. La IA busca automatizar esto.

RD-2Intermedio

El problema MAPF (Multi-Agent Path Finding) es NP-completo. ¿Qué significa esto? ¿Por qué es un problema de razonamiento deductivo?

NP-completo: No se conoce algoritmo de tiempo polinomial. El problema de decidir si existe solución de costo ≤ k es NP-completo (Nebel, 2019). Esto implica que en el peor caso, el tiempo de resolución crece exponencialmente con el tamaño.

MAPF como deducción: Dados un mapa (predicados de adyacencia), posiciones iniciales/finales de cada agente, y restricciones de tiempo (no colisión), el problema es deducir si existe una secuencia de movimientos (una "prueba") que satisfaga todas las restricciones. El set de premisas Σ = {mapa, posiciones, restricciones}, y la conclusión α = "existe plan de costo ≤ k".

RD-3Difícil

Ordena de menor a mayor complejidad: EXP, NP, PSPACE, 2-EXP. ¿Cuál es la relación de inclusión entre estas clases?

Orden: NP ⊆ PSPACE ⊆ EXP ⊆ EXPSPACE ⊆ 2-EXP

ClaseRecursoEjemplo en IA

Se sabe que P ⊊ EXP (distinción estricta probada). Las demás inclusiones se conjeturan estrictas pero no están probadas (P vs NP es uno de los Problemas del Milenio).

Definiciones clave (de la Guía de Ejercicios)

ConceptoDefinición
Átomop(c₁,...,cₙ) con p predicado de aridad n y cᵢ constantes
ReglaHead :- Body. Si Body⊆M → Head∩M≠∅
HechoRegla con Body vacío: p(a). equivale a {p(a)} :- {}

Sintaxis Clingo

% Hechos
gato(tom). perro(rex). humano(jorge).

% Reglas (Head :- Body)
mortal(X) :- humano(X).       % X en mayúscula = variable
amigos(X,Y) :- conoce(X,Y), conoce(Y,X).

% Regla disyuntiva (Head tiene >1 átomo)
gato(X) ; perro(X) :- animal(X).

% Constraint (elimina modelos)
:- gato(X), perro(X).         % nada puede ser gato Y perro

% Cardinality constraint
1{color(X,rojo);color(X,azul)}1 :- objeto(X).  % exactamente 1 color

% Negación por defecto (not)
libre(X) :- celda(X), not ocupado(X).

% Minimización
#minimize{1,X : asignado(X, equipo_mono)}.

% Show
#show color/2.

Reducción del programa (ΠX)

Para verificar si X es answer set, se calcula el programa reducido ΠX:

  1. Para cada regla con not a en el cuerpo: si a ∈ X, eliminar la regla; si a ∉ X, eliminar not a del cuerpo.
  2. ΠX no tiene negación. X es answer set si X es modelo minimal de ΠX.

ASP-4BásicoGuía ASP Ej.4

Para el programa:

p :- r.
t.

Demuestra que {p,t} y {t} son modelos, pero solo {t} es answer set.

¿{p,t} es modelo? Chequeamos cada regla:

  • Regla 1: {p:-r}. Body={r}. r ∉ {p,t}, así que Body ⊄ M. La regla no se activa — vacuamente satisfecha. ✓
  • Regla 2: {t:-∅}. Body=∅ ⊆ {p,t}. Head={t}, t ∈ {p,t}. ✓

{p,t} es modelo. ✓

¿{t} es modelo?

  • Regla 1: r ∉ {t}, no se activa. ✓
  • Regla 2: t ∈ {t}. ✓

{t} es modelo. ✓

¿{p,t} es answer set? No, porque {t} ⊊ {p,t} y {t} también es modelo. {p,t} no es minimal.

¿{t} es answer set? Debemos verificar que no hay subconjunto propio de {t} que sea modelo. Subconjuntos propios: {∅}. La regla {t:-∅} requiere que ∅ satisfaga t ∈ M, pero t ∉ ∅. Así que ∅ no es modelo. Por lo tanto {t} es minimal → es answer set.

¿Todo modelo debe contener t? Sí. La regla t:- tiene body vacío, por lo que siempre se activa. Todo modelo M debe satisfacer t ∈ M.

ASP-7IntermedioGuía ASP Ej.7

Demuestra que si el programa contiene la regla :- p, entonces ningún answer set M contiene p.

La regla :- p se escribe como {} :- {p}.

Por la definición de modelo: si p ∈ M (body ⊆ M), entonces Head ∩ M ≠ ∅. Pero Head = ∅, entonces ∅ ∩ M = ∅, que nunca es ≠ ∅.

Por lo tanto, ningún conjunto M que contenga p puede ser modelo del programa. En particular, ningún answer set (que es un modelo minimal) puede contener p. ∎

ASP-8IntermedioGuía ASP Ej.8

Demuestra que si el programa contiene la regla {} :- {} (constraint siempre activo), no tiene ningún answer set. ¿Puede tener modelos?

La regla {} :- {} tiene body = ∅ ⊆ M para todo M (siempre activa) y Head = ∅. Para ser modelo se necesitaría ∅ ∩ M ≠ ∅, lo cual es falso para cualquier M.

No tiene modelos (ni siquiera el conjunto vacío lo es, porque ∅ ∩ ∅ = ∅, nunca ≠ ∅).

Como no tiene ningún modelo, en particular no tiene ningún modelo minimal → no tiene answer sets.

¿Puede tener modelos? No. La constraint viola la definición de modelo para absolutamente todo conjunto M. En Clingo, esto se representa como :- . (body vacío, sin cuerpo) y hace que el programa sea insatisfactible (UNSATISFIABLE).

ASP-9DifícilGuía ASP Ej.9

Supón que Π tiene múltiples modelos minimales. Compara los modelos minimales de Π∪{:-p, q} versus Π∪{:-p}∪{:-q}.

Π∪{:-p, q}: La constraint es {} :- {p,q}. Body={p,q}. Se activa solo si p∈M y q∈M simultáneamente. Elimina los modelos minimales de Π que contienen AMBOS p y q.

Π∪{:-p}∪{:-q}: Dos constraints separadas. {} :- {p} elimina modelos con p. {} :- {q} elimina modelos con q. Elimina los modelos que contienen p (aunque no tengan q) y los que contienen q (aunque no tengan p).

Diferencia: Π∪{:-p, q} solo bloquea modelos con p∧q. Π∪{:-p}∪{:-q} bloquea modelos con p, y también modelos con q. Es mucho más restrictivo: p Y q están prohibidos cada uno individualmente.

Ejemplo: si M={p,r} es modelo minimal de Π, Π∪{:-p,q} lo conserva (p∈M pero q∉M). Π∪{:-p}∪{:-q} lo elimina (porque p∈M).

Dado el programa:

abuelo(X,Y) :- progenitor(X,Z), progenitor(Z,Y).

(a) ¿Cuántas reglas tiene el programa instanciado? (b) Agrega reglas para hermano, primo_hermano, nieto. (c) Explica la diferencia entre las 2 versiones de clausura transitiva.

(b) Relaciones adicionales:

nieto(X,Y) :- progenitor(Y,Z), progenitor(Z,X).
hermano(X,Y) :- progenitor(Z,X), progenitor(Z,Y), X != Y.
primo_hermano(X,Y) :- progenitor(Px,X), progenitor(Py,Y),
                       hermano(Px,Py), X != Y.

(c) Clausura transitiva:

Versión 1: alcanzable(X,Y) :- alcanzable(X,Z), alcanzable(Z,Y) — usa alcanzable en el body. Cuando se instancia, puede generar muchas más reglas: todas las combinaciones de pares alcanzables × alcanzables. Puede generar el mismo hecho por múltiples caminos.

Versión 2: alcanzable(X,Y) :- vuelo(X,Z), alcanzable(Z,Y) — siempre "avanza" al menos un vuelo. El programa instanciado es más pequeño (solo vuelo×alcanzable, no alcanzable×alcanzable). Ambos computan la misma clausura transitiva, pero la Versión 2 tiene menos reglas instanciadas.

Los modelos minimales son iguales en ambas versiones (misma extensión de alcanzable).

   not en(robot, _, T).  % debe estar en algún lugar
:- exec(recoger(robot,Obj), T),
   en(robot, P, T), not en(Obj, P, T).  % obj debe estar aquí

% (b) Efectos de acciones
en(robot, Dest, T+1) :- exec(mover(robot,Dest), T).
tiene(robot, Obj, T+1) :- exec(recoger(robot,Obj), T).

% (c) Persistencia (frame problem): lo que no cambia, permanece
en(X, P, T+1) :- en(X, P, T), 
                  not exec(mover(X, _), T),
                  paso(T).
tiene(robot, Obj, T+1) :- tiene(robot, Obj, T),
                           not exec(soltar(robot,Obj), T),
                           paso(T).

% Goal: robot en p3 con obj1
:- horizonte(T), not en(robot, p3, T).
:- horizonte(T), not tiene(robot, obj1, T).

La clave es la persistencia: en ASP, lo que no se afirma explícitamente como cambiado, se mantiene igual (frame axioms).
